# Growwlytics

AI-powered content intelligence for Instagram creators. Connect your account, sync your posts, and get a data-driven content plan generated by Claude — including hook ideas, a script outline, caption, hashtags, and optimal posting time.

---

## What it does

- **Connect Instagram** — OAuth via Facebook Graph API; tokens encrypted at rest (AES-256-CBC)
- **Sync posts** — pulls your last 50 posts with per-post engagement metrics (likes, comments, reach, impressions, saves)
- **Generate a content plan** — Claude analyzes your niche, audience, and post history, then returns a structured plan covering topic rationale, script segments, caption, and hashtag strategy
- **Track results** — compares posts made after a plan was generated against your 30-day baseline
- **Credits system** — 2 free credits on signup; buy more via Razorpay (₹99–₹1199)

## API routes

| Method | Path | Description |
|---|---|---|
| `GET` | `/health` | Health check |
| `GET` | `/api/auth/instagram` | Start Instagram OAuth (accepts `?token=<jwt>`) |
| `GET` | `/api/auth/instagram/callback` | Facebook OAuth callback |
| `GET` | `/api/credits` | Get current credit balance |
| `POST` | `/api/credits/purchase` | Create a Razorpay order |
| `POST` | `/api/credits/verify` | Verify payment and credit account |
| `GET` | `/api/instagram/status` | Check if Instagram is connected |
| `POST` | `/api/instagram/sync` | Sync last 50 posts + insights |
| `POST` | `/api/plans/generate` | Generate a content plan (costs 1 credit) |
| `GET` | `/api/plans/:id` | Get a saved plan |
| `GET` | `/api/plans/:id/results` | Get post performance vs. baseline |

All routes except `/health` and `/api/auth/*` require a `Authorization: Bearer <supabase_jwt>` header.

## Database schema

9 tables, all with row-level security (every row locked to its owning user):

`profiles` → `connected_accounts` → `posts`
`profiles` → `content_briefs`
`profiles` → `credits` + `credit_transactions`
`profiles` → `competitors` → `competitor_posts`
`profiles` → `sync_logs`

Two Postgres RPCs called by the API with the service-role key:
- `spend_credit(user_id, plan_id)` — atomic debit with row lock, raises `insufficient_credits` if balance is 0
- `add_credits(user_id, amount, ...)` — credits account and inserts ledger entry

## Running locally

```bash
# 1. Install dependencies
npm install

# 2. Copy env template and fill in values (see SETUP.md)
cp .env.example .env

# 3. Build shared types (required before the API can start)
npm run build --workspace=packages/types

# 4. Start frontend + backend in parallel
npm run dev
```

- Frontend → http://localhost:3000
- API → http://localhost:4000
- Health check → http://localhost:4000/health

## Environment variables

See [.env.example](.env.example) for the full list with descriptions.

Key variables:

| Variable | Used by | Purpose |
|---|---|---|
| `SUPABASE_URL` | API | Supabase project URL |
| `SUPABASE_SERVICE_ROLE_KEY` | API | Bypasses RLS for server-side writes |
| `NEXT_PUBLIC_SUPABASE_URL` | Web | Same URL, exposed to browser |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Web | Public anon key for Supabase JS client |
| `ANTHROPIC_API_KEY` | API | Claude content plan generation |
| `IG_APP_ID` / `IG_APP_SECRET` | API | Facebook App credentials |
| `IG_REDIRECT_URI` | API | Must point to `localhost:4000`, not `3000` |
| `RAZORPAY_KEY_ID` / `RAZORPAY_KEY_SECRET` | API | Payment processing |
| `ENCRYPTION_KEY` | API | AES-256-CBC key for token storage (64 hex chars) |
| `JWT_SECRET` | API | Signs OAuth state tokens |
| `APP_URL` | API | Frontend URL for post-OAuth redirects |
